3.1 Introdução

A efetividade das estratégias de manutenção não depende apenas da sua execução técnica, mas também de diversos fatores moderadores que influenciam os resultados. Este capítulo examina os principais fatores que afetam o desempenho da manutenção em caminhões fora de estrada.

3.2 Gestão da Manutenção

3.2.1 Estrutura Organizacional

A organização da função de manutenção impacta diretamente a efetividade:

3.2.1.1 Modelos de Gestão

  1. Manutenção Centralizada
    • Equipe única atende toda a operação
    • Maior especialização técnica
    • Possível aumento no tempo de resposta
  2. Manutenção Descentralizada
    • Equipes dedicadas por área/frota
    • Resposta mais rápida
    • Possível duplicação de recursos
  3. Modelo Híbrido
    • Manutenção de rotina descentralizada
    • Reparos complexos centralizados
    • Equilíbrio entre especialização e agilidade

3.2.2 Sistemas de Informação

3.2.2.1 CMMS (Computerized Maintenance Management System)

Funcionalidades essenciais:

  • Gestão de ordens de serviço: Criação, priorização, atribuição, fechamento
  • Controle de estoque: Peças, ferramentas, consumíveis
  • Histórico de equipamentos: Registro de todas as intervenções
  • Planejamento: Programação de preventivas e paradas
  • Indicadores: Dashboards de performance

3.2.3 Gestão de Estoque de Peças

3.2.3.1 Classificação ABC

Classificação de itens por valor e criticidade:

  • Classe A (20% dos itens, 80% do valor)
    • Componentes de alto valor
    • Gestão rigorosa de estoque
    • Exemplo: motores, transmissões
  • Classe B (30% dos itens, 15% do valor)
    • Componentes de valor intermediário
    • Gestão moderada
  • Classe C (50% dos itens, 5% do valor)
    • Componentes de baixo valor, alto volume
    • Gestão simplificada
    • Exemplo: filtros, porcas/parafusos

3.2.3.2 Estratégias de Estoque

Item Estratégia Estoque Lead Time
Crítico + Alto Valor Consignação Mínimo Reduzido
Crítico + Baixo Valor Estoque Local Alto N/A
Não-Crítico Just-in-Time Mínimo Aceitável

3.2.4 Capacitação da Equipe

Competências essenciais:

  1. Técnicas: Diagnóstico, reparo, uso de ferramentas especializadas
  2. Analíticas: Interpretação de dados, análise de falhas
  3. Sistêmicas: Compreensão de interdependências entre sistemas
  4. Digitais: Uso de CMMS, sistemas de diagnóstico eletrônico

3.3 Análise de Dados Recorrentes

3.3.1 Eventos Recorrentes em Manutenção

Diferente de análises de confiabilidade tradicionais que consideram tempo até primeira falha, a manutenção de equipamentos envolve eventos recorrentes:

  • Múltiplas falhas do mesmo tipo
  • Múltiplas intervenções preventivas
  • Substituições de componentes reparáveis

3.3.2 Modelagem de Processos Recorrentes

3.3.2.1 Processo de Poisson Homogêneo (HPP)

Assume taxa de falha constante:

\[ \lambda(t) = \lambda \]

Número esperado de eventos no intervalo \([0, t]\):

\[ \Lambda(t) = \lambda t \]

Limitação: Não captura envelhecimento ou melhoria após reparos.

3.3.2.2 Processo de Poisson Não-Homogêneo (NHPP)

Permite taxa de falha variável no tempo:

\[ \Lambda(t) = \int_0^t \lambda(s) \, ds \]

3.3.2.2.1 Modelo de Lei de Potência (Power Law Process)

\[ \lambda(t) = \frac{\beta}{\eta} \left(\frac{t}{\eta}\right)^{\beta-1} \]

Onde: - \(\beta > 1\): Deterioração (taxa crescente) - \(\beta < 1\): Melhoria (taxa decrescente) - \(\beta = 1\): Taxa constante (HPP)

Code 
library(ggplot2)

# Parâmetros
eta <- 1000
t <- seq(0, 3000, by = 10)

# Calcular taxa para diferentes betas
dados_taxa <- data.frame(
  t = rep(t, 3),
  beta = rep(c(0.8, 1.0, 1.5), each = length(t)),
  taxa = c(
    (0.8/eta) * (t/eta)^(0.8-1),
    (1.0/eta) * (t/eta)^(1.0-1),
    (1.5/eta) * (t/eta)^(1.5-1)
  )
)

dados_taxa$beta_label <- factor(
  dados_taxa$beta,
  levels = c(0.8, 1.0, 1.5),
  labels = c("β = 0.8 (Melhoria)", "β = 1.0 (Constante)", "β = 1.5 (Deterioração)")
)

ggplot(dados_taxa, aes(x = t, y = taxa, color = beta_label)) +
  geom_line(linewidth = 1.2) +
  labs(
    title = "Taxa de Falha - Processo de Lei de Potência",
    x = "Tempo (horas)",
    y = "Taxa de Falha λ(t)",
    color = "Parâmetro β"

3.3.3 Tipos de Reparo

3.3.3.1 Reparo Mínimo (Minimal Repair)

  • Restaura equipamento à condição imediatamente anterior à falha
  • Não altera a taxa de falha subsequente
  • Modelo: NHPP sem reinício

3.3.3.2 Reparo Perfeito (Perfect Repair)

  • Restaura equipamento a condição “como novo”
  • Reinicia o processo de envelhecimento
  • Modelo: Processo de Renovação

3.3.3.3 Reparo Imperfeito (Imperfect Repair)

  • Melhora condição, mas não totalmente
  • Redução parcial da idade efetiva
  • Modelos: Redução de idade virtual, Redução de intensidade

3.3.4 Estimação de Parâmetros

3.3.4.1 Método de Máxima Verossimilhança

Para NHPP com Lei de Potência, dados \(n\) eventos nos tempos \(t_1, t_2, \ldots, t_n\) em um período \([0, T]\):

Log-verossimilhança:

\[ \ell(\beta, \eta) = n \ln \beta - n \beta \ln \eta + (\beta - 1) \sum_{i=1}^{n} \ln t_i - \left(\frac{T}{\eta}\right)^\beta \]

Estimadores:

\[ \hat{\beta} = \frac{n}{\sum_{i=1}^{n} \ln(T/t_i)} \]

\[ \hat{\eta} = T \cdot n^{-1/\hat{\beta}} \]

Code 
# Função para estimar parâmetros do NHPP (Lei de Potência)
estimar_nhpp <- function(tempos_falha, tempo_observacao) {
  n <- length(tempos_falha)

  # Estimador de beta
  soma_log <- sum(log(tempo_observacao / tempos_falha))
  beta_hat <- n / soma_log

  # Estimador de eta
  eta_hat <- tempo_observacao * n^(-1/beta_hat)

  # Intervalo de confiança para beta (aproximado)
  se_beta <- beta_hat / sqrt(n)
  ic_beta <- beta_hat + c(-1.96, 1.96) * se_beta

  return(list(
    beta = beta_hat,
    eta = eta_hat,
    ic_beta = ic_beta,
    n_eventos = n,
    tempo_obs = tempo_observacao
  ))
}

# Exemplo: Falhas de um caminhão em 5000 horas
falhas <- c(450, 920, 1580, 2100, 2890, 3650, 4200, 4780)

resultado <- estimar_nhpp(falhas, 5000)

cat("Estimativas do NHPP:\n")

3.4 Política de Manutenção Preventiva Periódica

3.4.1 Conceito

Manutenção preventiva periódica consiste em intervenções programadas em intervalos fixos de tempo ou uso, independentemente da condição do equipamento.

Objetivos: - Reduzir probabilidade de falhas - Aumentar vida útil de componentes - Melhorar previsibilidade operacional

3.4.2 Disponibilidade e Mantenabilidade

3.4.2.1 Disponibilidade

A disponibilidade \(A\) é a fração do tempo em que o equipamento está disponível para operar:

\[ A = \frac{MTBF}{MTBF + MTTR} \]

Ou, considerando preventivas:

\[ A = \frac{Tempo_{Operando} + Tempo_{Espera}}{Tempo_{Total}} \]

3.4.2.2 Mantenabilidade

Mantenabilidade \(M(t)\) é a probabilidade de que um reparo seja completado no tempo \(t\):

\[ M(t) = P(T_{reparo} \leq t) \]

Para distribuição exponencial dos tempos de reparo:

\[ M(t) = 1 - e^{-\mu t} \]

Onde \(\mu = 1/MTTR\) é a taxa de reparo.

3.4.2.3 Relação com Manutenção Preventiva

Manutenções preventivas afetam:

  • MTBF: Pode aumentar ao prevenir falhas
  • MTTR: Preventivas geralmente mais rápidas que corretivas
  • Disponibilidade: Efeito líquido depende do equilíbrio

3.4.3 Confiabilidade e Periodicidade Ótima

3.4.3.1 Função Objetivo

A periodicidade ótima da preventiva minimiza custos ou maximiza disponibilidade.

Custo total por unidade de tempo:

\[ C(T_p) = \frac{C_p \cdot R(T_p) + C_f \cdot [1 - R(T_p)]}{\int_0^{T_p} R(t) \, dt} \]

Onde: - \(C_p\): Custo da preventiva - \(C_f\): Custo da falha (corretiva) - \(R(T_p)\): Confiabilidade no intervalo \(T_p\) - Denominador: Tempo médio de ciclo

3.4.3.2 Condição de Otimalidade

A derivada deve ser zero:

\[ \frac{d C(T_p)}{d T_p} = 0 \]

Simplificando (para Weibull):

\[ \frac{h(T_p^*)}{C_f/C_p - 1} = \frac{1 - R(T_p^*)}{\int_0^{T_p^*} R(t) \, dt} \]

Onde \(h(t) = \lambda(t)\) é a taxa de falha.

3.4.3.3 Algoritmo de Busca

Code 
# Função para calcular custo por unidade de tempo
custo_preventiva <- function(Tp, beta, eta, C_prev, C_falha) {
  # Confiabilidade em Tp
  R_Tp <- exp(-(Tp/eta)^beta)

  # Integral da confiabilidade de 0 a Tp (tempo médio de ciclo)
  integrand <- function(t) exp(-(t/eta)^beta)
  integral_R <- integrate(integrand, 0, Tp)$value

  # Custo por unidade de tempo
  custo <- (C_prev * R_Tp + C_falha * (1 - R_Tp)) / integral_R

  return(custo)
}

# Parâmetros exemplo
beta <- 2.3
eta <- 1500
C_prev <- 5000
C_falha <- 25000

# Busca do intervalo ótimo
intervalos <- seq(300, 1800, by = 10)
custos <- sapply(intervalos, custo_preventiva,
                 beta = beta, eta = eta,
                 C_prev = C_prev, C_falha = C_falha)

Tp_otimo <- intervalos[which.min(custos)]
custo_otimo <- min(custos)

cat("Intervalo ótimo de preventiva:", Tp_otimo, "horas\n")

3.4.3.4 Sensibilidade aos Parâmetros

O intervalo ótimo é sensível a:

  1. Razão de custos \(C_f/C_p\): Maior razão → intervalos mais curtos
  2. Parâmetro de forma \(\beta\):
    • \(\beta > 1\): Preventiva é benéfica
    • \(\beta < 1\): Preventiva pode ser contraproducente
  3. Parâmetro de escala \(\eta\): Define escala de tempo dos intervalos

3.5 Medidas de Performance da Manutenção

3.5.1 Conceito de Impacto

Avaliar a efetividade da manutenção preventiva requer comparar cenários:

  • Com preventiva vs. Sem preventiva
  • Diferentes intervalos de preventiva
  • Antes vs. Depois de mudança na política

3.5.2 Mensuração de Impacto na Incidência de Corretivas

3.5.2.1 Taxa de Falhas Observada

\[ \lambda_{obs} = \frac{Número_{Falhas}}{Tempo_{Exposição}} \]

3.5.2.2 Comparação de Taxas

Razão de Taxas (Rate Ratio):

\[ RR = \frac{\lambda_{com\_prev}}{\lambda_{sem\_prev}} \]

  • \(RR < 1\): Preventiva reduz falhas
  • \(RR = 1\): Sem efeito
  • \(RR > 1\): Preventiva piora situação (indicando sobre-manutenção ou outros problemas)

3.5.2.3 Modelo de Regressão de Poisson

Para ajustar por covariáveis (idade do equipamento, condições operacionais):

\[ \log(\lambda_i) = \beta_0 + \beta_1 \cdot Preventiva_i + \beta_2 \cdot Idade_i + \ldots \]

Code 
library(MASS)  # para glm.nb (binomial negativa)

# Dados exemplo (simulados)
set.seed(123)
n_equipamentos <- 50

dados <- data.frame(
  equip_id = 1:n_equipamentos,
  intensidade_prev = runif(n_equipamentos, 0, 10),  # preventivas/ano
  idade_anos = runif(n_equipamentos, 0, 10),
  horas_operacao = runif(n_equipamentos, 3000, 6000)
)

# Simular número de falhas (menor com mais preventivas)
dados$n_falhas <- rpois(n_equipamentos,
                        lambda = exp(2 - 0.15 * dados$intensidade_prev +
                                     0.08 * dados$idade_anos))

# Modelo de Poisson
modelo <- glm(n_falhas ~ intensidade_prev + idade_anos + offset(log(horas_operacao/1000)),
              data = dados,
              family = poisson(link = "log"))

summary(modelo)

# Interpretação
exp(coef(modelo))  # Rate Ratios

3.5.3 Mensuração de Impacto na Disponibilidade

3.5.3.1 Diferença de Disponibilidades

\[ \Delta A = A_{com\_prev} - A_{sem\_prev} \]

Ou em termos percentuais:

\[ \Delta A_{\%} = \frac{A_{com\_prev} - A_{sem\_prev}}{A_{sem\_prev}} \times 100\% \]

3.5.3.2 Análise de Séries Temporais

Disponibilidade pode variar ao longo do tempo. Modelos de séries temporais permitem:

  • Detectar tendências
  • Avaliar sazonalidade
  • Comparar períodos com diferentes políticas

3.5.3.3 Teste de Hipóteses

H₀: Não há diferença na disponibilidade média

H₁: Há diferença significativa

3.5.3.4 Análise de Covariância (ANCOVA)

Para controlar efeitos de variáveis confundidoras:

\[ Disponibilidade = \beta_0 + \beta_1 \cdot Política + \beta_2 \cdot Idade + \beta_3 \cdot Demanda + \epsilon \]

3.6 Resumo

🔔 Pontos-chave:

  • Gestão eficaz requer organização, sistemas de informação e equipes capacitadas
  • Eventos recorrentes exigem modelos específicos (NHPP, processos de renovação)
  • Periodicidade ótima de preventiva equilibra custos e confiabilidade
  • Impacto da manutenção deve ser mensurado com métodos estatísticos robustos
  • Disponibilidade e incidência de falhas são métricas complementares

## Análise de Dados Recorrentes {#sec-analise-dados-recorrentes}

### Eventos Recorrentes em Manutenção

Diferente de análises de confiabilidade tradicionais que consideram tempo até primeira falha, a manutenção de equipamentos envolve **eventos recorrentes**:

- Múltiplas falhas do mesmo tipo
- Múltiplas intervenções preventivas
- Substituições de componentes reparáveis

### Modelagem de Processos Recorrentes

#### Processo de Poisson Homogêneo (HPP)

Assume taxa de falha constante:

$$
\lambda(t) = \lambda
$$

Número esperado de eventos no intervalo $[0, t]$:

$$
\Lambda(t) = \lambda t
$$

**Limitação**: Não captura envelhecimento ou melhoria após reparos.

#### Processo de Poisson Não-Homogêneo (NHPP)

Permite taxa de falha variável no tempo:

$$
\Lambda(t) = \int_0^t \lambda(s) \, ds
$$

##### Modelo de Lei de Potência (Power Law Process)

$$
\lambda(t) = \frac{\beta}{\eta} \left(\frac{t}{\eta}\right)^{\beta-1}
$$

Onde:
- $\beta > 1$: Deterioração (taxa crescente)
- $\beta < 1$: Melhoria (taxa decrescente)
- $\beta = 1$: Taxa constante (HPP)

```{r}
#| label: fig-nhpp-taxa
#| fig-cap: "Função de taxa de falha do NHPP para diferentes valores de β"
#| warning: false

library(ggplot2)

# Parâmetros
eta <- 1000
t <- seq(0, 3000, by = 10)

# Calcular taxa para diferentes betas
dados_taxa <- data.frame(
  t = rep(t, 3),
  beta = rep(c(0.8, 1.0, 1.5), each = length(t)),
  taxa = c(
    (0.8/eta) * (t/eta)^(0.8-1),
    (1.0/eta) * (t/eta)^(1.0-1),

)

  geom_line(linewidth = 1.2) +
  labs(
    title = "Taxa de Falha - Processo de Lei de Potência",
    x = "Tempo (horas)",

    color = "Parâmetro β"

  scale_color_manual(values = c("#27AE60", "#95A5A6", "#E74C3C")) +
```

### Tipos de Reparo

#### Reparo Mínimo (Minimal Repair)

- Restaura equipamento à condição imediatamente anterior à falha
- Não altera a taxa de falha subsequente
- Modelo: **NHPP sem reinício**

#### Reparo Perfeito (Perfect Repair)

- Restaura equipamento a condição "como novo"
- Reinicia o processo de envelhecimento
- Modelo: **Processo de Renovação**

#### Reparo Imperfeito (Imperfect Repair)

- Melhora condição, mas não totalmente
- Redução parcial da idade efetiva
- Modelos: **Redução de idade virtual**, **Redução de intensidade**

### Estimação de Parâmetros

#### Método de Máxima Verossimilhança

Para NHPP com Lei de Potência, dados $n$ eventos nos tempos $t_1, t_2, \ldots, t_n$ em um período $[0, T]$:

Log-verossimilhança:

$$
\ell(\beta, \eta) = n \ln \beta - n \beta \ln \eta + (\beta - 1) \sum_{i=1}^{n} \ln t_i - \left(\frac{T}{\eta}\right)^\beta
$$

Estimadores:

$$
\hat{\beta} = \frac{n}{\sum_{i=1}^{n} \ln(T/t_i)}
$$

$$
\hat{\eta} = T \cdot n^{-1/\hat{\beta}}
$$

```{r}
#| label: exemplo-estimacao-nhpp
#| warning: false

# Função para estimar parâmetros do NHPP (Lei de Potência)
estimar_nhpp <- function(tempos_falha, tempo_observacao) {
  n <- length(tempos_falha)

  # Estimador de beta
  soma_log <- sum(log(tempo_observacao / tempos_falha))
  beta_hat <- n / soma_log

  # Estimador de eta
  eta_hat <- tempo_observacao * n^(-1/beta_hat)

  # Intervalo de confiança para beta (aproximado)
  se_beta <- beta_hat / sqrt(n)
  ic_beta <- beta_hat + c(-1.96, 1.96) * se_beta

  return(list(
    beta = beta_hat,
    eta = eta_hat,
    ic_beta = ic_beta,
    n_eventos = n,
    tempo_obs = tempo_observacao
  ))
}

# Exemplo: Falhas de um caminhão em 5000 horas
falhas <- c(450, 920, 1580, 2100, 2890, 3650, 4200, 4780)

resultado <- estimar_nhpp(falhas, 5000)

cat("Estimativas do NHPP:\n")
cat("β̂ =", round(resultado$beta, 3),
    "(IC 95%: [", round(resultado$ic_beta[1], 3), ",",
    round(resultado$ic_beta[2], 3), "])\n")
cat("η̂ =", round(resultado$eta, 1), "horas\n")

if (resultado$beta > 1) {
  cat("\nInterpretação: β > 1 indica DETERIORAÇÃO (taxa crescente)\n")
} else if (resultado$beta < 1) {
  cat("\nInterpretação: β < 1 indica MELHORIA (taxa decrescente)\n")
} else {
  cat("\nInterpretação: β ≈ 1 indica taxa CONSTANTE\n")
}
```

## Política de Manutenção Preventiva Periódica {#sec-politica-preventiva}

### Conceito

Manutenção preventiva periódica consiste em intervenções programadas em intervalos fixos de tempo ou uso, independentemente da condição do equipamento.

**Objetivos:**
- Reduzir probabilidade de falhas
- Aumentar vida útil de componentes
- Melhorar previsibilidade operacional

### Disponibilidade e Mantenabilidade {#sec-disponibilidade-mantenabilidade}

#### Disponibilidade

A disponibilidade $A$ é a fração do tempo em que o equipamento está disponível para operar:

$$
A = \frac{MTBF}{MTBF + MTTR}
$$

Ou, considerando preventivas:

$$
A = \frac{Tempo_{Operando} + Tempo_{Espera}}{Tempo_{Total}}
$$

#### Mantenabilidade

Mantenabilidade $M(t)$ é a probabilidade de que um reparo seja completado no tempo $t$:

$$
M(t) = P(T_{reparo} \leq t)
$$

Para distribuição exponencial dos tempos de reparo:

$$
M(t) = 1 - e^{-\mu t}
$$

Onde $\mu = 1/MTTR$ é a taxa de reparo.

#### Relação com Manutenção Preventiva

Manutenções preventivas afetam:

- **MTBF**: Pode aumentar ao prevenir falhas
- **MTTR**: Preventivas geralmente mais rápidas que corretivas
- **Disponibilidade**: Efeito líquido depende do equilíbrio

```{r}
#| label: fig-disponibilidade-vs-tp
#| fig-cap: "Impacto do intervalo de preventiva na disponibilidade"
#| warning: false
#| echo: false
#| eval: false

# Simulação do impacto
library(dplyr)

simular_disponibilidade <- function(Tp, beta, eta,
                                    tempo_prev, tempo_corr,
                                    tempo_sim = 8760) {
  # Simplificação: modelo determinístico
  n_preventivas <- floor(tempo_sim / Tp)
  tempo_em_preventivas <- n_preventivas * tempo_prev

  # Estimar falhas esperadas entre preventivas
  lambda_Tp <- (beta/eta) * (Tp/eta)^(beta-1)
  falhas_esperadas <- lambda_Tp * tempo_sim / Tp * n_preventivas
  tempo_em_corretivas <- falhas_esperadas * tempo_corr

  tempo_disponivel <- tempo_sim - tempo_em_preventivas - tempo_em_corretivas
  disponibilidade <- tempo_disponivel / tempo_sim

  return(disponibilidade)
}

# Parâmetros
beta <- 2.5
eta <- 2000
tempo_prev <- 8
tempo_corr <- 24

Tp_range <- seq(200, 1500, by 50)
disp <- sapply(Tp_range, simular_disponibilidade,
               beta = beta, eta = eta,
               tempo_prev = tempo_prev, tempo_corr = tempo_corr)

ggplot(data.frame(Tp = Tp_range, Disp = disp), aes(x = Tp, y = Disp)) +
  geom_line(linewidth = 1.2, color = "#3498DB") +
  geom_vline(xintercept = Tp_range[which.max(disp)],
             linetype = "dashed", color = "#E74C3C") +
  scale_y_continuous(labels = scales::percent) +
  labs(
    title = "Disponibilidade vs. Intervalo de Preventiva",
    x = "Intervalo de Preventiva (horas)",
    y = "Disponibilidade (%)"
  ) +
  theme_minimal()
```

### Confiabilidade e Periodicidade Ótima {#sec-periodicidade-otima}

#### Função Objetivo

A periodicidade ótima da preventiva minimiza custos ou maximiza disponibilidade.

**Custo total por unidade de tempo:**

$$
C(T_p) = \frac{C_p \cdot R(T_p) + C_f \cdot [1 - R(T_p)]}{\int_0^{T_p} R(t) \, dt}
$$

Onde:
- $C_p$: Custo da preventiva
- $C_f$: Custo da falha (corretiva)
- $R(T_p)$: Confiabilidade no intervalo $T_p$
- Denominador: Tempo médio de ciclo

#### Condição de Otimalidade

A derivada deve ser zero:

$$
\frac{d C(T_p)}{d T_p} = 0
$$

Simplificando (para Weibull):

$$
\frac{h(T_p^*)}{C_f/C_p - 1} = \frac{1 - R(T_p^*)}{\int_0^{T_p^*} R(t) \, dt}
$$

Onde $h(t) = \lambda(t)$ é a taxa de falha.

#### Algoritmo de Busca

```{r}
#| label: otimizacao-periodicidade
#| warning: false

# Função para calcular custo por unidade de tempo
custo_preventiva <- function(Tp, beta, eta, C_prev, C_falha) {
  # Confiabilidade em Tp
  R_Tp <- exp(-(Tp/eta)^beta)

  # Integral da confiabilidade de 0 a Tp (tempo médio de ciclo)
  integrand <- function(t) exp(-(t/eta)^beta)
  integral_R <- integrate(integrand, 0, Tp)$value

  # Custo por unidade de tempo
  custo <- (C_prev * R_Tp + C_falha * (1 - R_Tp)) / integral_R

  return(custo)
}

# Parâmetros exemplo
beta <- 2.3
eta <- 1500
C_prev <- 5000
C_falha <- 25000

# Busca do intervalo ótimo
intervalos <- seq(300, 1800, by = 10)
custos <- sapply(intervalos, custo_preventiva,
                 beta = beta, eta = eta,
                 C_prev = C_prev, C_falha = C_falha)

Tp_otimo <- intervalos[which.min(custos)]
custo_otimo <- min(custos)

cat("Intervalo ótimo de preventiva:", Tp_otimo, "horas\n")
cat("Custo esperado:", round(custo_otimo, 2), "$/hora\n")

# Visualizar
library(ggplot2)
ggplot(data.frame(Tp = intervalos, Custo = custos),
       aes(x = Tp, y = Custo)) +
  geom_line(linewidth = 1.2, color = "#2C3E50") +
  geom_point(x = Tp_otimo, y = custo_otimo,
             size = 4, color = "#E74C3C") +
  geom_vline(xintercept = Tp_otimo, linetype = "dashed", alpha = 0.5) +
  annotate("text", x = Tp_otimo + 150, y = custo_otimo,
           label = paste("Ótimo:", Tp_otimo, "h"), hjust = 0) +
  labs(
    title = "Otimização do Intervalo de Manutenção Preventiva",
    subtitle = paste("β =", beta, ", η =", eta, "| Cp =", C_prev, ", Cf =", C_falha),
    x = "Intervalo de Preventiva (horas)",
    y = "Custo por Hora de Operação ($/h)"
  ) +
  theme_minimal() +
  theme(plot.title = element_text(face = "bold"))
```

#### Sensibilidade aos Parâmetros

O intervalo ótimo é sensível a:

1. **Razão de custos $C_f/C_p$**: Maior razão → intervalos mais curtos
2. **Parâmetro de forma $\beta$**:
   - $\beta > 1$: Preventiva é benéfica
   - $\beta < 1$: Preventiva pode ser contraproducente
3. **Parâmetro de escala $\eta$**: Define escala de tempo dos intervalos

## Medidas de Performance da Manutenção {#sec-medidas-performance}

### Conceito de Impacto

Avaliar a efetividade da manutenção preventiva requer comparar cenários:

- **Com preventiva** vs. **Sem preventiva**
- **Diferentes intervalos** de preventiva
- **Antes** vs. **Depois** de mudança na política

### Mensuração de Impacto na Incidência de Corretivas {#sec-impacto-corretivas}

#### Taxa de Falhas Observada

$$
\lambda_{obs} = \frac{Número_{Falhas}}{Tempo_{Exposição}}
$$

#### Comparação de Taxas

**Razão de Taxas (Rate Ratio)**:

$$
RR = \frac{\lambda_{com\_prev}}{\lambda_{sem\_prev}}
$$

- $RR < 1$: Preventiva reduz falhas
- $RR = 1$: Sem efeito
- $RR > 1$: Preventiva piora situação (indicando sobre-manutenção ou outros problemas)

#### Modelo de Regressão de Poisson

Para ajustar por covariáveis (idade do equipamento, condições operacionais):

$$
\log(\lambda_i) = \beta_0 + \beta_1 \cdot Preventiva_i + \beta_2 \cdot Idade_i + \ldots
$$

```{r}
#| label: exemplo-regressao-poisson
#| warning: false
#| eval: false

library(MASS)  # para glm.nb (binomial negativa)

# Dados exemplo (simulados)
set.seed(123)
n_equipamentos <- 50

dados <- data.frame(
  equip_id = 1:n_equipamentos,
  intensidade_prev = runif(n_equipamentos, 0, 10),  # preventivas/ano
  idade_anos = runif(n_equipamentos, 0, 10),
  horas_operacao = runif(n_equipamentos, 3000, 6000)
)

# Simular número de falhas (menor com mais preventivas)
dados$n_falhas <- rpois(n_equipamentos,
                        lambda = exp(2 - 0.15 * dados$intensidade_prev +
                                     0.08 * dados$idade_anos))

# Modelo de Poisson
modelo <- glm(n_falhas ~ intensidade_prev + idade_anos + offset(log(horas_operacao/1000)),
              data = dados,
              family = poisson(link = "log"))

summary(modelo)

# Interpretação
exp(coef(modelo))  # Rate Ratios
```

### Mensuração de Impacto na Disponibilidade {#sec-impacto-disponibilidade}

#### Diferença de Disponibilidades

$$
\Delta A = A_{com\_prev} - A_{sem\_prev}
$$

Ou em termos percentuais:

$$
\Delta A_{\%} = \frac{A_{com\_prev} - A_{sem\_prev}}{A_{sem\_prev}} \times 100\%
$$

#### Análise de Séries Temporais

Disponibilidade pode variar ao longo do tempo. Modelos de séries temporais permitem:

- Detectar tendências
- Avaliar sazonalidade
- Comparar períodos com diferentes políticas

#### Teste de Hipóteses

**H₀**: Não há diferença na disponibilidade média

**H₁**: Há diferença significativa

Teste t para comparar médias de períodos:

```{r}
#| label: exemplo-teste-disponibilidade
#| warning: false

# Dados exemplo: disponibilidade mensal antes e depois de mudança
antes <- c(0.82, 0.84, 0.81, 0.83, 0.85, 0.84, 0.82, 0.83, 0.84, 0.83, 0.82, 0.84)
depois <- c(0.87, 0.88, 0.89, 0.86, 0.88, 0.87, 0.89, 0.88, 0.87, 0.88, 0.89, 0.87)

# Teste t
resultado_teste <- t.test(depois, antes, alternative = "greater")

cat("Disponibilidade média ANTES:", round(mean(antes), 3), "\n")
cat("Disponibilidade média DEPOIS:", round(mean(depois), 3), "\n")
cat("Diferença:", round(mean(depois) - mean(antes), 3), "\n")
cat("p-valor:", format.pval(resultado_teste$p.value), "\n")

if (resultado_teste$p.value < 0.05) {
  cat("\n✓ Diferença estatisticamente significativa (α = 0.05)\n")
} else {
  cat("\n✗ Diferença NÃO estatisticamente significativa (α = 0.05)\n")
}
```

#### Análise de Covariância (ANCOVA)

Para controlar efeitos de variáveis confundidoras:

$$
Disponibilidade = \beta_0 + \beta_1 \cdot Política + \beta_2 \cdot Idade + \beta_3 \cdot Demanda + \epsilon
$$
